Module 1

HTML Structure & Semantics

Building the Web's Backbone

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It is the literal "skeleton" of every page you see online.

The Document Tree (DOM)

HTML is hierarchical. Elements are nested like containers. Understanding the "Parent-Child" relationship is vital for CSS styling.

<header>
  <nav>
    <ul>
      <li>Home</li>
    </ul>
  </nav>
</header>

Tag Anatomy

Most tags have an opening and closing pair. Attributes live inside the opening tag to provide extra info.

<a href="url"> Link </a>
Tag Attribute Closing

The Power of Semantics

Semantic HTML means using tags that describe their meaning, not just their look. This is critical for SEO and Accessibility.

<header>

Top navigation and logo area.

<main>

The unique primary content.

<section>

Group related content items.

<footer>

Copyright and contact links.

Text Formatting

  • <h1-h6>

    Headings: Establishes visual hierarchy.

  • <p>

    Paragraph: The standard block for text body.

  • <strong>

    Bold: Indicates strong importance.

Lists & Links

  • <ul> & <ol>

    Unordered (bullet) vs Ordered (numbered) lists.

  • <li>

    List items that live inside containers.

  • <a>

    Anchor tags that create clickable links.

CSS Styling & Layouts

Designing the Digital Realm

CSS (Cascading Style Sheets) is the layer that brings beauty and life to the web. It controls the colors, fonts, spacing, and the overall layout of every element.

The Cascade Rule

If multiple rules apply to the same element, the most specific one or the one written last will win. This is why it's called "Cascading".

Modern Layout: Flexbox

Gone are the days of complex floats. Flexbox is the professional way to align items vertically and horizontally within a container.

Avoid Errors: Pro Tips

  • 1

    Validate your Tags: Always ensure every opening tag like <div> has a matching </div>. Unclosed tags are the #1 cause of broken layouts.

  • 2

    File Name Rules: Use only lowercase letters and hyphens (e.g., about-me.html). Never use spaces or special characters in filenames.

  • 3

    The "Console" is your friend: If something isn't working, check the 'Console' tab in DevTools. It will literally tell you the error line number.

Writing "Beautiful" Code

  • 1

    Consistent Indentation: Use 2 or 4 spaces consistently. Code that "steps" inwards is readable; flat code is a nightmare to fix.

  • 2

    Meaningful Classes: Use classes like btn-primary instead of blue-box-right. Describe the element's purpose, not just its current color.

  • 3

    Add Comments: Use <!-- Module End --> to mark sections. Future-you will thank you when you return to the code months later.
